# 📖 GitHub 프로필 대시보드 사용법

> 세계 최고의 선생님처럼 쉽게 설명합니다!

## 🎯 이게 뭐예요?

GitHub 프로필에 **자동으로 업데이트되는 진행상황 카드** 2개가 표시됩니다:

1. **프로필 카드** (원형 차트)
   - GitHub 커밋 통계 (1,476개 커밋)
   - 가입일 (2024년 8월)
   - 하루 평균 커밋 수 (3.3개)

2. **주간 대시보드** (6개 메트릭)
   - 코드 커밋, 유저 대화, 소셜 포스팅
   - 커피챗, 운동, 블로그 포스팅

## 🤖 어떻게 작동하나요?

### 자동화 시스템 3개

#### 1️⃣ GitHub 통계 카드 (매시간 자동 업데이트)
```
GitHub Actions (매시간)
   ↓
GitHub API에서 커밋 통계 가져오기
   ↓
SVG 카드 생성
   ↓
자동 커밋 + GitHub에 푸시
   ↓
README에서 바로 보임!
```

**내가 할 일**: 없음! 알아서 됩니다 ✨

#### 2️⃣ 주간 대시보드 (Slack으로 매일 업데이트)
```
오전 7시: Slack 리마인더 도착 📬
   ↓
나: "1 0 0 2 0 0 1 1" 입력
   ↓
GitHub Actions 실행
   ↓
data.json 업데이트
   ↓
대시보드 SVG 재생성
   ↓
README 히스토리 테이블 업데이트
   ↓
자동 커밋 + 푸시
   ↓
Slack 확인 메시지 도착 ✅
```

**내가 할 일**: 하루에 한 번, 8개 숫자만 입력!

#### 3️⃣ 주간 자동 리셋 (매주 월요일 오전 7시)
```
GitHub Actions (매주 월요일 7시)
   ↓
새로운 주 감지
   ↓
지난주 데이터 → weeklyHistory에 저장
   ↓
히스토리 SVG 생성 (dashboard/history/)
   ↓
README 히스토리 테이블 업데이트
   ↓
현재 주 메트릭 → 0으로 리셋
   ↓
자동 커밋 + 푸시
   ↓
새로운 주 시작! 🎉
```

**내가 할 일**: 없음! 완전 자동! ✨

**히스토리 보기**: README의 "Weekly History" 섹션에서 지난 12주 클릭해서 확인

## 📥 Slack 입력 방법

### 간단 버전: 8개 숫자만 입력
```
1 0 0 2 0 0 1 1
```

### 각 숫자의 의미
```
1      0      0      2           0             0          1       1
│      │      │      │           │             │          │       │
IG     TT     HT     유저대화    커피챗         블로그     러닝    헬스장
```

### 예시

**오늘 한 일**:
- Instagram 포스트 2개
- 유저 인터뷰 3번
- 헬스장 1번

**입력**:
```
2 0 0 3 0 0 0 1
```

**결과**:
- Slack에서 확인 메시지 받음
- 5분 후 GitHub README 업데이트됨
- 새로운 총계 표시됨

## 🎨 GitHub README에서 보는 법

### 데스크탑
- 카드들이 화면 중앙에 예쁘게 정렬됨
- 클릭하면 더 크게 볼 수 있음

### 모바일
- 자동으로 화면 크기에 맞춰짐
- 스크롤해서 전체 확인 가능

## 🛠️ 수동으로 업데이트하려면?

### GitHub Actions 직접 실행
```bash
# 프로필 카드 업데이트
gh workflow run profile-summary-cards.yml

# 대시보드 업데이트
gh workflow run update_dashboard.yml

# Slack 없이 직접 숫자 입력
gh workflow run slack_response.yml -f metrics="1 0 0 2 0 0 1 1"
```

## 📊 메트릭 설명

### 🚀 Code Commits (코드 커밋)
- **자동 계산**: git 히스토리에서 이번 주 커밋 수
- **목표**: 하루 20개

### 💬 User Talks (유저 대화)
- 유저 인터뷰, 피드백 세션, 사용자 미팅
- **목표**: 하루 1번

### 📱 Social Posts (소셜 포스팅)
- Instagram, TikTok, HelloTalk 합계
- 콘텐츠 마케팅, 브랜드 빌딩

### ☕ Coffee Chats (커피챗)
- 동료 창업자, 투자자, 멘토 미팅
- **목표**: 주 2회

### 🏃 Workouts (운동)
- 러닝 + 헬스장 합계
- 건강 관리, 스트레스 해소

### 📝 Blog Posts (블로그 포스팅)
- AI, 스타트업, 개발 관련 글
- 지식 공유, 브랜드 빌딩

## ❓ 자주 묻는 질문

### Q1: Slack 입력을 깜빡했어요!
**A**: 괜찮아요! 나중에라도 입력하면 됩니다. 숫자는 **누적**되니까 걱정 마세요.

### Q2: 프로필 카드에 더미 데이터가 보여요 (156 커밋)
**A**: 로컬에서 테스트할 때 정상입니다. GitHub Actions가 실행되면 실제 데이터(1,476 커밋)로 업데이트됩니다.

### Q3: 주간 총계는 언제 리셋되나요?
**A**: 매주 월요일 오전 7시에 자동으로 리셋됩니다! 지난주 데이터는 히스토리에 자동 저장되고, README에서 클릭해서 볼 수 있습니다.

### Q4: 커밋 수가 0으로 나와요
**A**: git 히스토리가 없거나 이번 주에 커밋이 없는 경우입니다. 코드를 푸시하면 자동으로 업데이트됩니다.

### Q5: Slack 리마인더가 두 번 왔어요
**A**: `data.json`의 `lastUpdated`가 오늘 날짜로 업데이트되지 않았을 수 있습니다. 한 번 입력하면 다음 리마인더는 안 옵니다.
